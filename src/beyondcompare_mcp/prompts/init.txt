# Beyond Compare MCP Initialization

This MCP server provides file and directory comparison capabilities using Beyond Compare. Below are the available tools and their usage:

## Available Tools

### 1. compare_files
Compare two files and optionally generate a diff report.

**Parameters:**
- `left_path` (str): Path to the left file
- `right_path` (str): Path to the right file
- `output_report` (str, optional): Path to save the HTML diff report

**Example:**
```python
result = await client.compare_files(
    left_path="/path/to/file1.txt",
    right_path="/path/to/file2.txt",
    output_report="/path/to/diff.html"
)
```

### 2. compare_folders
Compare two folders and optionally generate a diff report.

**Parameters:**
- `left_path` (str): Path to the left folder
- `right_path` (str): Path to the right folder
- `output_report` (str, optional): Path to save the HTML diff report
- `include_subfolders` (bool, optional): Whether to include subfolders (default: True)

**Example:**
```python
result = await client.compare_folders(
    left_path="/path/to/folder1",
    right_path="/path/to/folder2",
    output_report="/path/to/folder_diff.html",
    include_subfolders=True
)
```

### 3. sync_folders
Synchronize two folders with various sync modes.

**Parameters:**
- `source_path` (str): Source folder path
- `target_path` (str): Target folder path
- `sync_mode` (str): Sync mode ('mirror', 'update', or 'backup')
- `dry_run` (bool, optional): If True, perform a dry run without making changes (default: True)

**Example:**
```python
result = await client.sync_folders(
    source_path="/path/to/source",
    target_path="/path/to/target",
    sync_mode="mirror",
    dry_run=True
)
```

## Configuration

The server can be configured using environment variables or command-line arguments:

```bash
# Environment variables
export BEYOND_COMPARE_PATH="/path/to/bcompare"
export LOG_LEVEL="INFO"

# Or command line arguments
beyondcompare-mcp --host 0.0.0.0 --port 8000 --log-level DEBUG
```

## Error Handling

All tools return a dictionary with the following structure:
```python
{
    "success": bool,      # Whether the operation succeeded
    "message": str,       # Human-readable message
    "data": dict,        # Operation-specific data
    "error": str or None # Error message if operation failed
}
```

## Best Practices

1. Always use absolute paths
2. Check the `success` field in the response before accessing `data`
3. Use `dry_run=True` for sync operations to preview changes
4. Handle exceptions using try/except blocks

## Example Workflow

```python
# Initialize client
client = await create_mcp_client("beyondcompare-mcp")

# Compare two files
try:
    result = await client.compare_files(
        left_path="/docs/file1.txt",
        right_path="/docs/file2.txt",
        output_report="/reports/diff.html"
    )
    if result["success"]:
        # Use structured logging instead of print
        import logging
        logger = logging.getLogger(__name__)
        logger.info(f"Comparison complete. Report saved to {result['data']['report_path']}")
    else:
        logger.error(f"Error: {result['error']}")
except Exception as e:
    logger.error(f"Error comparing files: {str(e)}")
```

## Troubleshooting

- If Beyond Compare is not found, set the `BEYOND_COMPARE_PATH` environment variable
- Check the server logs for detailed error messages
- Ensure the server has read/write permissions to the specified paths
